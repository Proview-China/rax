# Application执行底座P0前置收口

## 本次闭合

- governed Operation新增持久Domain Reservation水位：`intent_recorded→domain_reserved→effect_admitted`；Domain Owner在任何Runtime mutation前原子保留exact rev1 Attempt、Intent、Subject、Session和Candidate；
- Reservation回包丢失只按attempt Inspect；Inspect回值必须Validate并与原请求完全一致。历史过期Fact仍可读取用于恢复/终止判断，但Admission、Issue、Begin、Declare、Prepare和Execute前复读发现过期即关闭；
- 新增Runtime Binding currentness只读桥。Application先验证Runtime投影，再生成不超过30秒且不超过Runtime expiry的本地授权；它不产生Binding、续租、dispatch或Commit资格；
- Run创建、Lifecycle、Start Confirmation和certified Claim V3保持同一Plan Certification Association；Application不创建生产Certification Fact，也不成为Certification Owner；
- CAS成功回值、lost-reply Inspect和Create/Inspect Fact全部Validate；只接受exact请求或从同一水位可证明的严格append-only successor。并发Sibling跨多阶段前进可回到外层状态机，等revision、换Plan、换Certification与换sidecar均拒绝；
- 两个未知namespaced未来模块通过同一Router和Coordinator完成Reservation→Runtime→Observation→Settlement，不增加kind switch；跨Owner、换Adapter、Descriptor漂移、授权撤销/过期全部fail closed；
- fake深拷贝补齐StartConfirmation内嵌Attempt/Settlement/Delegation，测试读取方不能修改存储值。
- Runtime Binding currentness先验证expected Binding；Governed Completion先验证Plan和Attempt ID；Recovery List/Acquire/Release新增集中public request Validate，非法Scope、游标、Journal/Owner、Policy、revision、epoch、TTL及TTL溢出均在backend前拒绝。
- 修正post-prepared unknown的Runtime水位衔接：持久Enforcement使Permit Fact从Begin单调`+1`时，仅当Prepared和Enforcement精确绑定同一effect/permit/attempt/operation/provider且`RecordedRevision`等于unknown revision才接受；无来源、错引用、同revision或大于`+1`均拒绝。

## 边界

这些结果闭合的是Application公共执行底座、恢复语义和测试替身，不选择生产数据库、RPC、消息队列、Scheduler、可信Plan Certification后端或SLA。未来6+1及更多用户自定义模块只能通过namespaced Descriptor、精确Binding、Domain Reservation和稳定Port接入，不得导入Application/Runtime内部实现或自授执行资格。
