# Route Gateway组合层完成

## 事件

- 时间：2026-07-11 14:30 CST
- 范围：上游调用与统一封装最终候选B阶段
- 结果：同一Go module内的`routegateway`组合包已完成离线实现和独立验收

## 已完成

1. `Gateway`固定执行Policy/Audit预检 → RuntimeBindingResolver → SecretResolver → AdapterPool/Factory → Provider；Policy失败时后四层计数均为0。
2. `SecretMaterial`只由类型化CredentialReference解析产生，格式化永久脱敏；Pool key只含非秘密轮换版本，不含秘密值、摘要或请求内容。
3. `RuntimeBinding`锁定Route七维身份、Deployment和Region，只允许解析Project/Workspace/Resource/DeploymentName引用；Factory具体Endpoint必须保持Catalog scheme/host/base path。
4. 14个现有Runtime AdapterID都有内建工厂；39条callable Route均完成真实Adapter构造并执行实际Route级Capabilities，无默认ADC/Entra/AWS链和公网调用。
5. AdapterPool按Route identity/evidence、Credential Version、Binding Version和Factory版本复用；实现并发单飞、轮换失效、旧Lease释放后Close和Gateway Close。
6. Gateway提供Resolve、Capabilities、Invoke、Stream、Close；流在终态或Close前持有Lease。
7. Azure legacy与Vertex Chat/Messages由Factory回报Adapter实际协议路径，修复Catalog基路径与实际Binding路径不相等时的审计错配。

## 约束

- 根包`RouteInvoker`仍是Policy/Authorization/Audit层；`routegateway.Gateway`才拥有秘密解析、Adapter构造和生命周期，两者不再混称。
- `CatalogBindingResolver`只用于纯构造/验证环境；生产宿主必须注入真实的非秘密BindingResolver。
- 本阶段没有实现模型自动选择、Profile、Prompt、Context、缓存策略、工具执行或Runtime状态机。
