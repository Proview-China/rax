## 关键审查修正

- Request或PreparedPlan封装后被修改会在上游接触前拒绝；Request/Plan IntentGraph、ExpectedProfile/ExpectedRoute不能分叉；
- ModelEvent、ExecutionItem、Control、Tool Effect与Header的Action/Item/Attempt/Approval身份强关联；
- Executed、ResultOrigin、SyntheticReason组合fail closed，synthetic结果不能生成Effect；
- camelCase、snake_case、连字符和嵌套credential key均清洗；已知秘密字面量在事件入账前脱敏；
- Effect必须关联已接受Intent和对应Attempt/Plan Intent；Verification不得悬空；supersession只能引用更早Effect；
- Runtime与Projector使用plan-bound EventLedger：Adapter不能接受或发布密封PreparedPlan之外的Intent/MechanismPlan，也不能用同ID篡改Plan内容；
- fallback图必须引用存在的同Intent Mechanism，且拒绝重复、自环和任意环；
- RuntimePolicy路径集合使用语义交集，`/workspace`到`/workspace/subdir`的合法收紧不会误变为空；
- 命令/取消派发结果不确定、重复terminal candidate、Manifest陈旧digest、未知工具调用均转为明确拒绝或`indeterminate`。
- Qwen/Claude双向stream-json取消命令增加发布屏障：`cancelPending`在接收侧解码快速`control_response + result`前必定可见，避免终态抢在取消确认与静止证据前提交。
