# Profile 统一语义路由定位确认

- 时间：2026-07-12 18:35:51 CST
- 模块：`model-invoker`
- 阶段：计划草案修订，等待用户继续共同设计

## 本次确认

1. Profile的核心目的不是保存配置，而是按精确模型、Offering和执行面，把Praxis并集语义双向转换为API、SDK、CLI、App Server或ACP的原生语义；
2. Harness中介会带入厂商指令、settings、skills、tools、会话和工作区语义，不能视为纯净模型；
3. Praxis先利用官方配置点和扩展点准备Harness，必要时在许可证、条款、版本和身份边界允许的范围内采用wrapper、窄补丁或sidecar；
4. OpenCode、OpenClaw只作为实现参考，不继承它们的OAuth身份、allowlist资格或厂商授权；
5. 新增`SemanticProfile`，负责request encoder、event decoder、能力映射、损失策略和命名空间扩展；
6. 上层通过`UnifiedExecutionRequest`与带类型的`UnifiedExecutionEvent`并集调用，不需要理解不同Harness私有语义；
7. 无法完全消解的差异必须进入`MappingReport`、`ContextEnvelopeReport`和`CapabilityResiduals`，不得静默丢失。
