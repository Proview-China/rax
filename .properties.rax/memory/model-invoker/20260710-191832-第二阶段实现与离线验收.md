## 终审修复

- 三家客户端全部拒绝 HTTP 3xx；同域 302、跨域 307、HTTPS 降级 308 的第二跳请求数均为 0。
- 3xx 在读取 body 前关闭并拒绝；巨大 3xx 的 body 读取次数为 0。
- 三家非流与流式响应统一使用 8 MiB 解压后硬上限；超限错误码为 `response_body_limit_exceeded`、不可重试、不保留 URL/SDK/secret cause。
- 非流调用捕获 status/header/body 并恢复下游 Body；流式调用只捕获 status/header，不预读 SSE body。
- Response、Error、StreamEvent、State、NativeEvents、Raw、Metadata、MappingReport 和格式化输出统一深度清洗 API Key 的原文、JSON 转义和 URL 转义形式。
- TextDelta、ReasoningDelta、ArgumentsDelta 使用跨事件状态化清洗；可能构成 secret 的后缀延迟输出，终态或异常 EOF 前安全 flush，公开 Stream 在读取前、pending 中和终态后的格式化均不暴露 secret。
- malformed 2xx 保留脱敏 RawResponse，SDK/transport 错误对象不进入公开 unwrap 链。
- Anthropic Prompt Cache 创建能力改为 Unsupported；provider options 与 continuation 均不能借 `cache_control` 绕过。
- Anthropic continuation 严格验证 thinking signature、redacted data、tool 字段和可续接内容；流强制唯一 `message_start`、合法 message/block 生命周期，非 direct tool caller 在产生统一函数事件前拒绝。
- Gemini 无 ID 与有 ID 工具快照都确定性去重；后到 thought signature 会合并，签名冲突明确拒绝；伪造的多语义 continuation 无法下发 SDK。
- Gemini 父 Context 取消、异常 EOF 和 provider error 均保留已完成的 Raw、NativeEvents、工具调用与 continuation。
- Gemini 在 tool → result → 普通文本后继续返回防御性 State，下一轮可完整重放必要的 tool/thought/signature 历史；无历史纯文本仍保持无状态。
- Anthropic continuation 只生成并接受规范化的 signed thinking、redacted thinking 和 direct tool_use；图片、文档、server tool、未知字段、结构层缓存控制和非 direct caller 均在 HTTP 前拒绝，工具业务 input 中同名字段不受误伤。
