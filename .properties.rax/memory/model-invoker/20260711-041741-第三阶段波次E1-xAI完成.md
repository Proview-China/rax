## 官方合同结论

1. 当前批准路线只使用 `grok-4.5 + Responses API + https://api.x.ai/v1 + XAI_API_KEY`；
2. Responses是推荐路径，Chat Completions已是 legacy；旧模型重定向会改变实际模型和价格，因此不进入本切片；
3. `grok-4.5` reasoning只允许 `low/medium/high`且不能禁用；
4. 默认状态保存30天并以 `previous_response_id`继续；`prompt_cache_key`用于缓存粘性路由；
5. 消费者 Grok Build产品没有转换为按量 API Credential，原控制记录继续不可调用。

## 实际产物

1. 新增 `provider/xai`，Provider ID为 `xai`，默认且唯一协议为 Responses；
2. 固定生产 Endpoint，BaseURL覆盖只允许 loopback测试；官方未给出可靠 Key前缀，因此没有编造前缀门禁；

5. reasoning禁用、summary/budget、结构化输出、strict函数、托管工具、旧模型、Chat、gRPC和其他未批准能力均拒绝或保持延期；
