## 本轮重要事实纠正

1. Qwen Code当前`--bare`会忽略`coreTools`并使用固定工具集；必须拆成`bare_fixed`与`controlled_nonbare`，禁止继续声明“bare + exact coreTools”。
2. Moonshot官方主项目已从旧Python Kimi CLI迁移到当前TypeScript Kimi Code。旧Agent spec、Wire与StrReplace行为只能归入`legacy_pinned`；当前Kimi Code使用ACP/stream-json、Edit/Write/Bash，公开CLI没有完整prompt override与精确tool registry allowlist。
3. Codex `dynamicTools`只是追加动态工具，不是全部Codex工具allowlist；stable app-server主要暴露Agent Thread/Turn/Item，不能伪造模型step。
4. Claude semantic-stable应使用Praxis最小桥接system prompt；`tools`控制可用性，`allowed_tools`只控制预授权。SDK继承父进程环境，必须固定resolved CLI并生成sanitized environment digest。
5. Gemini即使完整替换system prompt，仍有首用户`<session_context>`；它必须进入InjectionManifest。
