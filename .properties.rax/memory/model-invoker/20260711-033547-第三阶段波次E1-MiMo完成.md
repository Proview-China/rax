## 官方合同刷新

- 官方 `llms.txt`和 API Reference当前只公开 OpenAI Chat Completions与 Anthropic Messages，没有 Responses合同；因此只实现两条 Binding；
- 按量 Endpoint为 `api.xiaomimimo.com`，普通 Key使用 `sk-*`；Token Plan使用独立 `tp-*` Key和中国、新加坡、欧洲三类专属域名，两者不互换；
- 当前文本 callable模型只允许 `mimo-v2.5-pro`与 `mimo-v2.5`；2026-06-30已失效的 V2旧名、模态、Web Search、ASR/TTS均保持 slice外；
- 两协议都只把 portable reasoning effort映射为 thinking启用/禁用，不伪造推理深度；Chat JSON Object可用，严格 JSON Schema输出不在当前合同。

## 已落地产物

1. 新增 `provider/mimo/`独立 Runtime Adapter，默认 Messages，补充 Chat；只允许固定官方 HTTPS host或 loopback测试服务，拒绝 `tp-*`、Token Plan域名、重定向、错协议、未知模型和静默模型映射；
2. `internal/compatprovider`新增 Messages Bearer认证选择，MiMo不发送 Anthropic SDK默认的 `X-Api-Key`；
3. `internal/protocol/anthropicmessages`新增 Provider专属 stop reason扩展；MiMo把 `repetition_truncation`归为 incomplete/other，把 `content_filter`归为 policy rejected；
4. Messages保留 thinking/signature、工具 continuation与 parallel开关，并在发送前移除内部 `caller`；Chat保留非流/流 `reasoning_content`与 JSON Object；thinking工具历史无法保真时在 HTTP前拒绝；
5. 新增两条 `implemented_offline + callable=true` Catalog Binding。Catalog由55条扩展为57条：34条 callable、23条控制记录，对应十二个 Runtime Adapter；原六条 MiMo Token Plan控制记录仍无 Adapter且不可调用；
6. 新增 `tests/mimo/`黑盒与 fuzz，以及 `tests/integration/mimo_smoke_test.go`三重显式烟测入口；README、Matrix、module、properties和总计划已同步。
