## 安全终审修正

- Mantle SigV4服务名固定为 `bedrock-mantle`，与 Runtime的 `bedrock`签名域分离；本机测试直接检查 Credential scope。
- Bedrock bearer显式设置 `httpBearerAuth`优先级；TLS本机 SDK fake验证实际 `Authorization: Bearer`。
- Vertex Claude保留 Anthropic SDK官方 URL/body middleware，同时重新套回无跳转、响应捕获、8 MiB上限和受控认证 transport；本机测试验证模型移出 body、`vertex-2023-10-16`和 `rawPredict`路径。
- AWS原生错误按 context、HTTP状态与 Smithy错误码归一化；SDK对象、原生 cause和 provider message不进入公开 unwrap或错误文本。
- Bedrock、Vertex和 Azure的 OpenAI兼容路径均删除 SDK占位 Authorization，再注入云 Provider自己的认证；OpenAI Key不能作为 Bedrock Credential。
