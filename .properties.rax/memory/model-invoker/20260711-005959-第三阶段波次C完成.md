## 官方证据刷新

2026-07-11读取公开官方文档后，旧的“GLM、MiMo、Alibaba全面 `terms_blocked`”结论已失效：

- Kimi Code：`https://www.kimi.com/code/docs/en/`；允许会员 Key接入第三方 Coding Agent，要求保持真实 User-Agent，产品后端走开放平台。
- MiniMax Token Plan：`https://platform.minimax.io/docs/token-plan/intro`；提供独立 Token Plan Key并允许 Agent/Coding工具使用，不与 PAYG Key互换。
- GLM Coding Plan：`https://docs.bigmodel.cn/cn/coding-plan/quick-start`；明确专属 Coding Endpoint和受支持编程工具。
- Xiaomi MiMo Token Plan：`https://mimo.mi.com/docs/tokenplan/subscription`；提供 `tp-*` Key与三 Region Endpoint，禁止自动脚本、自定义应用后端和非 Coding API用途。
- Alibaba Coding Plan：`https://help.aliyun.com/en/model-studio/coding-plan`；提供 `sk-sp-*` Key与 Coding Endpoint，允许交互式编程工具并禁止后端、自动脚本和 API测试器。
- Alibaba Token Plan Team：`https://help.aliyun.com/en/model-studio/token-plan-quickstart`；允许编程工具/OpenClaw类 Agent，专属 Key/Endpoint与其他计划隔离。
- xAI消费者/Grok Build没有找到可供 model-invoker使用的公开订阅 API Key/Base URL合同，继续 `unverified + official_client_only`。

新结论只把前六类路线改为 `interactive_coding_only + planned + callable=false`，不授权伪装客户端、产品后端或真实调用。
