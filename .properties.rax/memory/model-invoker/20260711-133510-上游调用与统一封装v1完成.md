## 3. 固定行为

1. RouteID是唯一上游选择输入；调用方 Request中的 Provider、Protocol和 Endpoint必须为空。
2. 调用顺序固定为 Route存在、selector、callable/status、调用时 evidence、protocol/endpoint、static model、Offering/entitlement、禁止 PAYG、Adapter注册、基础 Invoker。
3. `Resolve`不调用 Provider任何方法。
4. non-callable、evidence过期、策略或 entitlement拒绝不会触达 Provider。
5. 有效订阅也保持 `AllowsAutomaticPAYGSwitch=false`，不会切换 Route、Key、账号或按量余额。
6. RouteSelection与基础 Response.MappingReport并列审计，不修改冻结的统一 Response/Stream语义。
7. 缓存只允许 ProviderOptions严格传输和 read/write usage归一化，不实现任何缓存策略。
