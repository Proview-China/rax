## 审计中确认的边界

### Continuation保留完整工具语义

State继续区分 native ID与确定性语义 ID，保存必须回传的 model/tool/result/text历史、thought signature和已响应索引。未知字段、不一致 call索引、伪造生成 ID、错误角色、不受控原生 Part和重复响应继续在 HTTP前拒绝；对外 State身份由 Binding注入。

### 迭代流不是简单 delta拼接

GenerateContent流可能重复完整 function-call快照，thought signature也可能在后续终态快照才出现。迁移保留原有按 native ID或 ID-less位置去重、参数指纹、后到签名合并、签名冲突拒绝和单调 Praxis事件序列，没有把它降级为普通文本流。

### Provider身份与安全错误

直接测试使用 `acme-generate`非原厂 Binding完成 thought → tool result → final response往返，证明 Response、State、MappingReport、流终态和错误身份均来自 Binding。SDK对象、Details map、HTTP request、Credential和任意 native cause不进入公开 unwrap链；Gemini details只提取白名单 Request ID、RetryInfo和 billing信号。
