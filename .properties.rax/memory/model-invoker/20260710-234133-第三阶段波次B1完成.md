## 审计中修复的问题

### HTTP 408重试语义

B1完整 HTTP错误矩阵发现 B0通用 Failure门禁会清除 HTTP 408的 Retryable标记。现已允许经过 Dialect分类的 `ErrorTimeout`保持 Retryable；context取消或截止路径仍不可重试，并只保留 `context.Canceled`或 `context.DeadlineExceeded`。

### 红后身份与 Endpoint

若 API Key恰好与 Provider ID相同，Redactor会正确清洗相同文本，但也可能误伤身份字段，因此最外层仍需 Binding恢复 Provider/Protocol。若原始 Endpoint碰巧含密钥片段，直接恢复真实 Binding又会把该片段带回公开 MappingReport。

现已分离：driver使用真实 Binding做安全选择；Adapter最外层使用只对 Endpoint做过脱敏的公开 Binding。反例同时证明 Provider/Protocol保持权威，公开 Endpoint不包含秘密。
