# 模型调用器：第一阶段实现与离线验收

- 时间：2026-07-10 15:44:00 CST
- 模块：`model-invoker`
- 阶段变化：第一阶段计划执行中 → 第一阶段离线实现完成

## 事件

`ExecutionRuntime/model-invoker/` 已完成第一阶段 Go 统一内核和 OpenAI Provider。Responses 与 Chat Completions 的非流式/流式链路均通过无密钥离线验收，计划转为陈旧计划。

## 已落地产物

1. Provider/Protocol/Endpoint/Model 分离的统一请求与响应；
2. 四级能力契约、字段级 reasoning summary/function error result 能力、确定性 MappingReport；
3. 并发安全 Registry、超时覆盖能力查询、Runtime 单点非流式重试、流不重放与终态关闭；
4. 统一错误、Request ID、Retry-After、受控 RawPayload 和失败审计；
5. OpenAI Responses 与 Chat Completions 的请求映射、响应归一化、工具、结构化输出、推理、状态和 usage；
6. Responses typed SSE 与 Chat chunk/delta 流；
7. 外部包 core/OpenAI 黑盒测试、固定协议样本、fuzz 种子、benchmark；
8. 默认排除的真实 OpenAI 烟雾测试入口；
9. 模块使用说明、`.properties.rax/module` 说明和项目总索引。

## 关键验收修复

- typed-nil Error/Stream 不再导致 panic；
- `Budget.Timeout` 覆盖 Capability 查询与 Provider 调用；
- 拒绝时收集全部能力决定，Response/Error/失败流均保留 MappingReport；
- Endpoint 进入 CapabilityQuery 与 MappingReport，Responses 返回明确续接 State；
- `Strict=nil` 保留协议默认，不再静默强制为 true；
- SDK API 错误和结构化流错误不穿透 SDK 类型或 Authorization；
- 2xx 畸形载荷不可重试，SDK 重试为 0，Retry-After 不被本地 backoff 上限缩短；
- Responses/Chat 终态后停止读取并关闭底层响应体；
- 流失败保留 RawRequest、RawResponse、累计 NativeEvents 和 Request ID；
- Chat effort-only 不需要降级，reasoning summary 和函数错误结果使用独立能力记录。
