# 2026-07-16 07:32 Tool Call候选观测v1完成

## 事件

Model Invoker完成`ToolCallCandidateObservationV1`与`ToolCallCandidateObservationProjectionV1`公共合同、direct sync/stream原子事件输出和严格Codec闭合。中央全门禁通过，独立Review最终结论为`YES`。

## 结果

- sync/stream只在完整`completed/tool_call`批次finalize后恰好构造并发出一次Projection事件；
- exact ref覆盖Contract、Invocation、Observation、Response ID、SourceSequence以及每个Call的ordinal、ID、name和canonical arguments；
- N>1保持整批，不允许部分升级；逐Call事件明确为compatibility-only，不是Gateway authority；
- 失败、取消、EOF、冲突、partial或lost terminal均零Projection；
- 终态Response ID必须匹配已绑定ID；空ID只能继承唯一绑定；无绑定拒绝；
- 公共Codec拒绝顶层和任意嵌套重复JSON key；
- targeted ordinary 100轮、race 20轮，以及中央full ordinary/race、vet、gofmt和diff均通过。

## 边界

该产物仍是Provider Observation，不是`PendingAction`、`ActionCandidate`、派发资格或执行授权。Harness/Tool/Runtime后续接线必须消费完整Projection，不能把逐Call兼容事件当作权威输入。
