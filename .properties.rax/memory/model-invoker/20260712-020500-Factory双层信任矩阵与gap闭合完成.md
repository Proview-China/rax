## 复核gap闭合

1. 自定义Factory返回`Provider(io.Closer)+buildErr`但漏填`FactoryResult.Closer`时，Gateway先从Provider推导Closer，再关闭一次；build与close原文不泄露，close cause保留`errors.Is`。
2. `adapterPool.acquire`在Build返回、候选发布前复查调用方context；忽略取消的Factory迟到成功结果会关闭一次，并同时保留cancellation/deadline与安全Close cause。
3. `Capabilities`/`Invoke`在Provider调用失败时安全合并stale lease `releaseErr`，Closer失败不再丢失。
4. Gateway二次响应模型错配的Stream Event先覆盖可信Provider/Protocol/Endpoint身份，清空Response/Raw；模型错配和显式`routegateway.Stream.Close`均用opaque safe cause保留`errors.Is`且不泄密。
