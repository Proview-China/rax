## 1. 用户新增的确定目标

第三阶段不能只继续增加厂商直连 Adapter，还必须覆盖：

- 各家 Token Plan、Coding Plan、会员权益和其他官方订阅；
- AWS Bedrock上的 Anthropic等云托管模型；
- OpenAI Responses/Chat Completions与 Anthropic Messages兼容路线；
- 官方没有 Go SDK时的非 Go官方 SDK/API；
- 有实际市场存量的第三方托管与自托管上游；
- 最终形成基本覆盖主流上游的统一调用体系。

## 2. 已固定的架构

完整身份改为：

```text
UpstreamRoute = Model Family
              + Provider
              + Offering
              + Deployment
              + Protocol
              + Endpoint
              + Credential Profile
```

关键决定：

1. 模型原厂、计费/服务 Provider、订阅产品、部署和协议分离；
2. 同一模型在直连、云托管和第三方托管下是不同 Route；
3. 按量 API与 Token/Coding Plan的 Key、Endpoint、配额和条款不得互换；
4. SDK所有权区分 Provider第一方、模型方、协议方、云平台和社区；
5. Go继续拥有统一语义、路由、能力、策略和审计；TypeScript/Python只作为隔离执行器；
6. `allowed_usage` 固定为 `general_api`、`interactive_coding_only`、`official_client_only`；
7. `terms_blocked` 路线可以建档，但不能注册为可调用 Provider；
8. 订阅额度不得隐式回退到 PAYG，不得伪造 User-Agent或官方工具身份。

## 3. 官方订阅调查结果

| 计划 | 当前判断 |
|---|---|
| GLM Coding Plan | 只允许官方支持工具，Praxis直连为条款阻塞 |
| Kimi Code | 只允许个人交互式 Coding/Agent，必须保留真实客户端身份；产品后端走开放平台 |
| MiniMax Token Plan | 允许自定义 Agent/Coding客户端；生产 SaaS/backend权利仍未知 |
| MiMo Token Plan | 禁止自动脚本、自定义应用后端和非 Coding API，Praxis直连为条款阻塞 |
| Alibaba Coding/Token Plan | 只允许交互式编程工具/Agent，通用后端为条款阻塞 |
| Alibaba Savings Plan | 只是按量 API费用抵扣，不是新 Provider或协议 |
| xAI消费者订阅 | 只确认 Grok Build登录/headless/ACP，没有订阅 API Key与 Base URL证据 |
| Claude Pro/Max | 与 Anthropic API、Bedrock、Vertex额度分离，不能转换为调用凭据 |

## 4. 云托管调查结果

- AWS Bedrock必须分开 Mantle与 Runtime：
  - Mantle提供 OpenAI Responses、Chat Completions和 Anthropic Messages；
  - Runtime提供 Converse/Invoke、Chat与 Messages，不提供 Responses；
  - Bedrock bearer、SigV4、Region、Project/Workspace和模型兼容表分别建模；
  - Mantle Responses默认保存30天，`store=false`后不能使用 `previous_response_id`。
- Vertex AI必须分开 Gemini原生、Claude Partner Model与 OpenAI兼容 Chat Endpoint；
- Azure OpenAI v1使用 Deployment name，支持 API Key/Entra，旧 `api-version`路线独立；
- Anthropic Go SDK当前已支持 Bedrock Mantle和 Vertex认证入口，不需要只为这两条路线增加 Sidecar；
- Anthropic Message Batches与 AWS/Google云 Batch是不同能力域；
- Claude Platform on AWS是 Anthropic运营、AWS Marketplace计费的另一产品，不是 Bedrock。
