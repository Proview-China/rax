## 粗粒度事件

独立第三审确认四个 P1，本轮均只在 Review 独占范围修复：

1. Store public shape 存在 Case-only create 绕过入口。修复后，`ports.StoreV1` 与 `memory.Store` 均删除 `CreateCaseV1`；仓内合法 Case 创建只经 `caseengine.Engine.CreateCaseV1` 组装并调用复合 Store mutation。
2. Verdict Decide 未检测 Decision current Reader 读期间的时钟回拨。修复后，在 `InspectDecisionCurrentV1` 前读取 non-zero baseline，在 Reader 返回后读取 fresh now；若 `now < baseline`，在 Verdict 构造和 Store Decide/CAS 前返回 ClockRegression。反例让 Reader 内部把时钟从 T2 回拨到 T1，且 T1 仍晚于全部 facts，验证仍为 zero Verdict/CAS。
3. Target history 缺少严格单调与不可覆盖约束。修复后，history 为 append-only `(Revision -> Digest)`：same revision换digest与revision rollback均Conflict且零写，新 revision严格大于全部current/history；旧exact Target在冲突后仍可读取。
4. Target+Case 与 optional Trace 缺少原子事务。修复后，统一使用 `CreateTargetCaseMutationV1{Target, Case, optional Trace}`：非空 Requested Trace 与 Target+Case 在同一锁、同一线性化点发布；Trace ID/source-sequence 冲突在首个写入前返回，Target/Case/新Trace三者零写，不使用失败后删除。lost reply 只 exact Inspect 原 Target、Case、Trace，mutation调用一次。

RunID 继续由 Target 与 current Policy exact 绑定；没有增加 TurnID，也没有自行发明 Turn 语义。

P2 测试补齐：Auto invalid matrix新增cross-tenant并逐项exact Inspect证明Attestation/Verdict零写；ReviewerID、ReviewerAuthority、ReviewerBinding逐项漂移均在Decide前zero CAS；Policy、ActorAuthority、ReviewerAuthority、Scope、Binding、两项Evidence逐个作为唯一最短TTL验证Verdict expiry，并在actual-point过期验证zero Verdict/CAS。
