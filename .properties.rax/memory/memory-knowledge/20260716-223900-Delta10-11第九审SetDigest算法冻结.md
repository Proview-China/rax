# Delta 10/11第九审Set Digest算法冻结

时间：2026-07-16 22:39 CST

## 问题

V2声明了Memory六个、Knowledge八个set digest字段，但此前没有逐项domain/version/ObjectKind、canonical body和正规化算法；实现无法唯一重算，OrderedItemSet还可能误用含fresh `ExpiresAt/Digest`的完整ProjectionItem。

## 修正

- 冻结两Owner独立set-digest domain/version与十四个ObjectKind。
- 冻结ordered item、content ref、exact ref、string digest四类canonical body；全部调用`contract.Digest(body)`，禁止`MustDigest`和map顺序。
- `OrderedItemSetDigest`只使用stable item body，排除item `ExpiresAt/Digest`，保持Owner canonical顺序与0-based Rank。
- Content按ID/Digest/Length/MediaType正规化；Ref集合使用`contract.NormalizeRefs`；Citation/License/Conflict digest按字节序升序compact。
- 空集合固定编码`[]`；所有set digest必须重算比对，不能只验证非空。
- 更新N71，新增N74与Conformance 27。

## 当前真值

该修正关闭第九审第三个Owner P1后，Owner-local为P0=0/P1=0/P2=0；External P0=5保持NO-GO。本轮未写Go。
