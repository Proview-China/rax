# QEMU/KVM MicroVM真实后端

时间：2026-07-18 12:46 CST

## 事件

Sandbox Rust Data Plane新增产品中立`microvm` Surface下的首个QEMU `microvm` + KVM Adapter。
kernel/initramfs只由root-owned opaque binding解析并逐内容SHA-256复核；Dispatch payload只携binding
ID/digest、vCPU、memory、network-deny-all与wall-clock bound，不接受路径、QEMU参数或kernel cmdline。

真实本机门使用`/dev/kvm`、QEMU 10.1和宿主6.17内核只读测试副本启动独立guest，静态BusyBox
initramfs输出`PRAXIS_MICROVM_OK`后退出。另一条长运行guest证明：Inspect读取running，Release在进程
存活时Conflict，Fence等待PID+`/proc` start-time identity消失后才返回fenced，Cleanup若进程仍存活
只能返回residual_present。64路不同artifact并发allocate只有一个稳定binding分支胜出；KVM降级、
artifact binding/digest漂移均fail closed。

## Owner边界

Provider仍只产生Observation/Receipt，不写Sandbox/Runtime Fact，不迁移Lease/Fence/Instance epoch。
公共Policy只使用`microvm` Surface，不写死QEMU。guest agent、块设备、Snapshot、Secret、network
allow-list、Restore与宿主部署认证仍未实现，不能因本事件宣称完整production能力。
