# 官方MCP普通取消Unknown语义实证

时间：2026-07-18 14:30 Asia/Shanghai

## 粗粒度结果

- live official MCP Go SDK v1.6.1确认普通request cancellation由`CallTool`的context触发
  `notifications/cancelled`；SDK仍没有可复用Task public nominal。
- Tool owner-local新增真实in-memory SDK Session Conformance：Call完成admission并进入Server
  handler后取消context，执行Entry必须进入Unknown且没有Protocol Receipt；同stable key再次提交
  只返回既有admission，不重复调用Provider。
- 定向ordinary×100与race×20通过。

## 边界

- 普通cancel notification可能被忽略，也可能与迟到响应竞争，不能生成原Effect
  `confirmed_not_applied`。
- Task取消继续等待官方public nominal并使用独立`tasks/cancel`；不得以普通cancel替代。
- 受治理Cancel/Close仍等待Runtime独立矩阵与actual-point Port；本事件只增加上游行为证据，
  不开放SDK/API/CLI写入口或production能力。
