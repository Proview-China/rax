# MCP受治理Connect领域闭环软件验收YES

时间：2026-07-18 11:20 +08:00

## 结果

- `praxis.mcp/connect`使用独立Run/Session矩阵，不复用`praxis.tool/execute`或单Action门。
- official MCP Go SDK的stdio与Streamable HTTP只在受控actual-point executor内构造；Connect Intent、create-once admission和lost-reply Inspect-only已经闭合。
- Protocol Receipt经Runtime正式Observation/Evidence后，由Tool Owner生成typed `MCPConnectDomainResultFactV1`；Runtime Settlement V4完成后，Tool才创建ApplySettlement并发布Connection Availability。
- transport-neutral API与CLI只提供Intent/Receipt/Connection/DomainResult/Apply/Availability exact Inspect，不提供Connect/Discover/Call写后门。

## 边界

这是owner-local软件闭环，不是production GO。production Credential/Network policy materializer、durable backend、Application/composition root与SLA仍不存在。Discovery每个分页请求仍需独立`praxis.mcp/discover` Runtime治理入口；该跨Owner实现尚未获本阶段授权，旧注入式Discovery只能用于owner-local测试。
