# PD-TM-04第七设计修正候选

时间：2026-07-16 19:48 Asia/Shanghai

## 事件

第六审后对live Application、Harness、Runtime与Tool字段进行交叉复核，发现三项不能由现有公共合同证明的语义：Application/Harness没有PendingAction Revision；Association/Generation没有Active Route三元组；Surface/Schema没有authoritative Payload LimitPolicy和完整InputSchema current来源。因此`PD-TM-04`重新进入第七设计修正候选，待用户确认与独立设计终审；不是冻结合同、不是YES。

## 候选内容

- PendingAction Revision候选固定为Tool-local wrapper revision `1`，跨Owner只证明ID+RequestDigest，禁止挪用其他对象Revision；
- Active Route只由Route current完整canonical projection和exact Reader证明，Association/Generation只交叉实际拥有的Generation与BindingSet五字段；
- 提出Tool Owner additive `ToolInputContractCurrentRefV1/ProjectionV1/ReaderV1`，权威闭合Surface entry、Capability、Tool、InputSchema、LimitPolicy与InputSchemaCurrent；
- 优先保持`ActionCandidateV2`不变，把Input Contract exact Ref/Projection加入Candidate current closure；当前没有live Validate迫使升级Candidate版本；
- S1 Resolve、S2与Boundary exact Inspect、immutable create-once lease、fresh clock、requested TTL、lost reply、typed-nil、nil context、nominal request和deep-copy全部列为待实现门禁；
- existing skeleton P0-a/b/c/d必须进入测试矩阵，不能用fixture自证或把施工骨架描述为完成。

## 当前门禁

Tool Go hard-block；P4/system/production NO-GO。不得修改现有施工骨架、调用Provider/Gateway、创建production root或进入P5。下一步是用户确认候选字段与Owner语义，再交第七次独立设计终审。
