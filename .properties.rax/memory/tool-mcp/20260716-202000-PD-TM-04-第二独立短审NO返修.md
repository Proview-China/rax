# PD-TM-04第二独立短审NO返修

时间：2026-07-16 20:20 Asia/Shanghai

## 结论

第二独立短审最终NO：P0=5/P1=2/P2=0。当前仅返修Tool自有设计资产，待用户确认与独立终审；不是冻结合同、不是YES。Tool Go hard-block，P4/system/production均NO-GO。

## 候选修正

- Surface由Tool-owned Repository EnsureExact/InspectExact，Capability/Tool由Registry Record+完整Descriptor证明；三条来源Owner独立，Route ExpectedOwner另行证明；
- Schema没有Owner；InputSchema Current Authority绑定exact Tool Registry Object Current Ref；
- Model SourceCandidate改为无TTL/无Owner的immutable historical exact ref，不新增Model Port；
- P4只接受ActionCandidateV3 + BindingCurrentV2；InputContract Ref、一个Surface Current、两个Registry Object Currents、historical Model与Payload/Policy/Schema进入Candidate canonical；
- Tool Owner在完整S1+S2和共同最小窗之后、任何Watermark前原子持久BindingV2 durable root；BindingV2 Ref是唯一恢复根；
- live旧Watermark不含BindingV2 Ref；P5只新增派生handoff proof，不成为第二Truth Owner；
- InputContract与BindingV2 InspectExact都返回同一immutable Projection/Checked/Expires；只有外部Surface Current和Capability/Tool Registry Object Currents可以重新读取并收紧Boundary deadline；
- Tool contract只保存Application Request/PendingAction/Scope等中立稳定坐标；Application V2 DTO仅由applicationadapter消费并逐字段转换，禁止contract层反向依赖Application；
- Registry Object Kind显式映射live裸`capability/tool`，Registry/Surface fresh Projection digest与各自source Ref digest分离；只有InputContract/BindingV2 immutable lease满足`Ref.Digest==ProjectionDigest`；
- Surface Invocation三Owner桥联合裁决NO（P0=2/P1=1）：Model需public `PreparedModelInvocationRefV1+PreDispatchGateV1`，Tool Binding需闭合Prepared/Surface/Assembly/Injection/Profile/RegistrySnapshot/Generation/Handoff/BindingSet/Created/NotAfter，Harness只组合adapter/wiring且Tool Ack前Provider=0，Application不改；P1覆盖所有Provider路径统一Gate。等待最终合同与用户授权，P4继续hard-block；
- 旧26项已建立`PD-TM04-N01..N26`唯一编号，schema专审8P0/3P1/1P2及跨Owner新增反例均列入候选。
