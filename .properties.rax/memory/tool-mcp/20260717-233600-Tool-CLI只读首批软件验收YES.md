# Tool CLI只读首批软件验收YES

## 事件

- 新增可嵌入CLI Runner V1，只实现`tool list`与`tool inspect`；
- `tool list`消费transport-neutral Catalog typed cursor，绑定单一Registry Snapshot，检测跨页漂移后保持零输出；
- `tool inspect`要求Kind+ID+Revision+Digest exact坐标并只调用SDK Inspect；
- `tool call`、`mcp discover`及所有未闭合命令确定性返回unsupported，不创建Candidate、Operation或Provider调用；
- CLI不读取`os.Args`/环境、不创建根二进制，不导入网络、进程、Application/Harness/Model/Runtime kernel或官方MCP SDK。

## 验证

- `go test ./cli ./tests/conformance -run 'RunnerV1|CLIV1' -count=100`：PASS；
- `go test -race ./cli ./tests/conformance -run 'RunnerV1|CLIV1' -count=20`：PASS；
- 64并发list输出一致；typed-nil、nil/canceled context、非法exact ref、未知命令和Snapshot漂移反例通过。

## 当前裁决

- CLI只读首批状态：`implementation_software_test_yes`；
- 根CLI、`call/cancel/watch/mcp discover/package verify|install|revoke`、Webhook与production装配仍未实现；
- 执行类命令必须等待统一Application/Runtime/Sandbox治理Port，不允许把SDK、官方MCP Session或CLI变成Provider后门。
