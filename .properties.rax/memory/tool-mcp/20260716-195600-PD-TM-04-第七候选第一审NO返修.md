## 审计结论

## 返修内容

- Input Contract新增独立Binding/Issuance Subject；Provider Binding、ExpectedOwner、完整Surface Entry、Capability、Tool/Artifact、Schema、内建LimitPolicy与requested bound全部进入issuance canonical和稳定ID；
- Ref ID只由Issuance digest派生，Ref Digest只等于immutable Projection digest；Resolve、InspectByIssuance、InspectExact与mandatory内部Store三口均为distinct nominal合同；
- Surface Entry digest复用`praxis.tool-mcp.surface + SurfaceContractVersion + ToolSurfaceEntry`源canonical；Entry必须完整持久且包含`praxis.tool/execute`；
- LimitPolicy固定为Tool内建`praxis.tool/input-payload-v1`并由typed subject派生，不再假设live存在可配置权威源；
- Projection签发Surface/Capability/Tool/InputSchema四项current；前三项保留各source独立S1 Checked/Expires，Projection Checked取读取完成后的fresh clock，Expires取三项source、15秒cap与requested的最小值，InputSchemaCurrent使用Projection时间；
- exact source拆为S1按Surface/Capability/Tool exact Ref+ExpectedOwner Resolve current与S2携S1完整OwnerCurrentRef Inspect exact current；S2不得刷新lease，禁止latest/name弱读与CandidateSource自报；
- 四项Kind固定且Owner统一ExpectedOwner；InputSchemaCurrent ID把ExpectedOwner纳入派生以隔离跨Provider碰撞，Revision=1，Digest等于Schema ContentDigest；
- Candidate Resolve与Binding lease保存完整Input Contract projection/closure；P4入口必须消费完整closure。Watermark/Boundary三Ref是后续P5 additive Delta，不冒充live字段；
- Input Contract独立TTL cap候选为15秒，不复用DomainResult 30秒；S1/S2/Boundary纳入最早expiry并保持Checked单调、短窗缩短、穿窗拒绝、长窗不延长；
- 第六审26项反例提供继承/替换/扩充映射；existing skeleton P0/P1、nominal、nil-context仍是未来代码验收项，不宣称已修。
