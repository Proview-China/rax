# SQLite + RocksDB生产存储闭环

时间：2026-07-17 15:22:00 +08:00

## 结论

Continuity owner-local生产持久化已实现。公共SPI保持Backend无关；这不解锁跨Owner Timeline current、Checkpoint capture、Restore、remote Provider、production root或SLA。

## 选择与实现

- SQLite候选以同一WAL/FULL/IMMEDIATE/busy-timeout Conformance比较：modernc纯Go约`43–45us/write`、`10.8–11.0us/read`；mattn/CGO约`34–35us/write`、`8.5–8.7us/read`。在当前无SLA且仓库已有modernc依赖的前提下，选择modernc以保持默认Go/CGO-disabled可构建；mattn只保留在显式benchmark tag。
- SQLite schema version 3支持additive初始化/升级，持久化Journal/Object/Retention、Timeline Event/Attempt/Policy/current以及Checkpoint Manifest/Seal history/current/CAS。Event历史与Tombstone overlay分离，所有owner current均由结构化scope key定位。
- 系统RocksDB 9.10经`librocksdb-dev`提供。第三方Go binding与9.10 C API发生版本漂移，最终改为`continuity_rocksdb` build tag下的极窄C API bridge；只实现ContentStore，不泄漏CGO类型。
- RocksDB实测5轮：4KiB sync Put约`32–33us`、校验Get约`6.2us`；1MiB sync Put约`4.6–5.0ms`、校验Get约`0.82–0.85ms`。这些是开发机基线，不是SLA。
- SQLite↔RocksDB在`metadata_pending | content_staged | reference_committed | visible`四个durable cut后关闭并重开，均只Inspect原Journal并收敛到同一closed Object。

## 仍未支持

- remote blob/archive/purge、KMS执行、自动orphan reclaim、在线backup/灾备与SLA；
- 真实typed Owner Readers、Application/Assembler生产root；
- Runtime/Harness/Application跨Owner Checkpoint与任何Restore执行；
- 外部世界回滚。Partial仍只诊断，Unknown/lost reply仍只Inspect原identity。
