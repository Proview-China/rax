# CLI/API治理写面范围冻结

时间：2026-07-18 01:24 CST

## 用户裁决

Continuity CLI/API V1选择“包含受治理写面”，不是只读首版。范围覆盖Timeline Projection、Checkpoint Create、Fork、Rewind Plan、Restore、Attach Artifact与Resolve Retention；只读Timeline/Checkpoint/Plan/Retention能力继续保留。

## 冻结边界

- 所有写命令只提交Application namespaced workflow，不能直写Continuity/Application Fact Store或Runtime legacy Command。
- caller只能提供stable Request/Idempotency、目标Scope、领域coordinate/plan exact ref、expected revision与not-after。
- caller不得提供可信current、Review Authorization、Permit/Fence、Provider binding/Receipt、Runtime Outcome或raw Application Submission Bundle。
- Submit或执行回包丢失只Inspect原Request/Command/Journal/Attempt；不得换ID重派。
- Restore、生产Checkpoint Participant、remote purge/archive等运行能力未闭合时返回unsupported且Provider=0；命令存在不代表能力启用。

## 最小公共Delta

live Application只有root实现层`FacadeV2.SubmitWorkflowV2`，没有组件可依赖的公开Submit+Inspect Port。A-CTY-01请求Application Owner在公开`contract/ports`增加版本化`ContinuityWorkflowSubmissionGatewayV1`，由trusted CompiledGraph/Binding/Descriptor装配Submission Bundle并返回Command/Plan/Outbox/Journal exact refs。Continuity不得私建同形接口或导入Application root实现。

## 当前状态

设计范围已冻结；Application公开Gateway与各kind Descriptor/Schema/Capability尚未联合Review/实现。因此Continuity Go写面仍为NO-GO，现有只读SDK不扩权。
