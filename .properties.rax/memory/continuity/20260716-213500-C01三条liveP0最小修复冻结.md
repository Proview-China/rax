## 2. live真值：一个umbrella P0，三个入口

| 入口 | live证据 | production缺口 |
|---|---|---|
| Project | `ExecutionRuntime/continuity/domain/timeline.go`的`Timeline.Project`把caller `EvidenceAdmission`转换为Event并调用`PutProjection` | caller可携可信bool/sequence/digest/Trust/payload语义；没有Attempt、Owner Readers S1/S2或current验证 |
| Rebuild | 同文件`Timeline.Rebuild`批量接受caller Candidate，转换后调用`ReplaceLedgerScope` | bulk caller import绕过Owner Readers、Attempt CAS与逐Item unknown恢复，并可覆盖Scope历史 |
| Tombstone | `ExecutionRuntime/continuity/storage/memory/backend.go`的`TombstoneProjection`原地修改Event的`Visibility/TombstoneRef` | 破坏Event revision/digest/canonical bytes不可变性，历史Inspect可能随current操作漂移 |

三者合计为`umbrella production P0=1（三入口）`。现有`PutProjection`、`ReplaceLedgerScope`、`TombstoneProjection`只能保留为reference/internal合同或反例，production Adapter不得装配。

## 3. 冻结的最小修复

1. production `TimelineProjectionRequestV1`字段闭集只有stable Attempt ID/idempotency key、`EvidenceSourceKey`、可选expected RecordRef、仅`authoritative_fact`可带OwnerFact ref、Projection Policy、Scope和`RequestedNotAfter`。禁止caller semantic/causal/object/payload/time、可信bool、sequence、digest、Trust或current。
2. Controller唯一序列为Create Attempt → S1 Record双读 → R-CTY-06 current/readability/tombstone → 仅`authoritative_fact`的Owner current → S2 exact复读 → fresh `ValidateCurrent` → 原子Event+Attempt visible+Continuity current index。Attempt状态包含`reconcile_required`；lost reply/unknown只Inspect原Attempt，禁止ABA。
3. Rebuild只接受Request闭集或Request ref；每个Item复用同一Attempt+S1/S2/fresh/atomic Controller，产生独立exact结果。禁止bulk caller Candidate/Event import、`ReplaceLedgerScope`、清空Scope、覆盖history/Tombstone overlay或回退current index。
4. Timeline Event create后revision/digest/canonical bytes永久immutable。Tombstone另建immutable revision 1 `TimelineProjectionTombstoneFactV1`，并与Visibility overlay/current index在Continuity Owner同一事务CAS；Query/Watch组合历史Event与overlay，历史Event零改写，删除overlay或Rebuild不得静默复活。
5. 六类`EvidenceTrustClassV2`保持Runtime Ledger语义：五类Observation形态原样投影；只有`authoritative_fact`追加typed Owner Fact current Reader。`RequestedNotAfter`不传入Owner Reader、不参与Owner重封或ProjectionDigest，只能最终缩短共同TTL。

## 4. 仍在外部的最小门

- `P1-1`：Runtime Evidence Owner联合终审并实现R-CTY-06 current/readability/tombstone/subject-current typed Reader；Evidence Ledger仍是sequence与Record唯一Owner。
- `P1-2`：Application/Assembler与各领域Owner冻结仅`authoritative_fact`使用的typed Owner Fact current Reader routing。
- Continuity不拥有Runtime Evidence identity/current、领域Fact、Runtime Outcome、Binding/Policy/Trust、Harness Slot/Phase或Application装配。
