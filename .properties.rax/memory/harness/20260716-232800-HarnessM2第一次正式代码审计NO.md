# Harness M2第一次正式代码审计NO

## 事件

Harness M2第一次正式代码审计结论为`NO(P0=2/P1=3)`。虽然target100、race20、full ordinary、full race与vet均已通过，现有`model_predispatch_assembly_current_v1`实现仍只保留为`candidate-NO-GO`，不得解锁Tool、Capability、system G6A或production root。

## P0

1. caller提供的Manifest、Conformance、ToolSurface与Profile只做shape验证后直接进入Seal，未由verified Compile/Manifest/Conformance与ToolSurface Owner current Reader证明；
2. 没有独立BindingSet current Reader，只读取`GenerationBindingAssociationFactV1.Candidate.Binding`历史嵌套Projection，无法发现BindingSet换代或撤销。

## P1

1. UnknownOutcome recovery只有caller deadline/Owner TTL约束，缺少独立host hard cap；
2. 真实ABA CAS、64个不同内容、same-ID drift、Reader post-clock cancellation、tamper等对抗证据不完整；
3. Handoff current Seal会静默覆盖错误非零ContractVersion。

## 裁决

- 停止用更多同类测试替代Owner-current P0；
- 保留现有代码与已绿测试，不回滚、不标完成；
- 先完成[Model PreDispatch Assembly Owner-current Port Delta](../../design/harness/port-deltas/model-predispatch-assembly-owner-current-v1.md)的联合裁决；
- Runtime neutral verified Assembly Inputs、ToolSurface current、BindingSet current公共合同及对应Owner Reader落盘、compile、审计YES后，Harness才可返修M2并重新提交独立代码审计。
