# Harness G6A跨模块测试组合候选

## 事件

Harness新增首条N=1 G6A跨模块测试组合，使用公开合同串接真实Harness `SingleCallToolActionAssemblerV2`/InputCurrent Reader、Application `SingleCallToolActionCoordinatorV2`和Tool `SingleCallToolActionAdapterV2`，闭合到settled ToolResult与current Runtime V4 Inspection/Association。

该事件仅是`test-only composition candidate`，不宣称system G6A或生产能力。

## 已证明

- Application Request/InputCurrent由Harness五Reader及S1/S2路径产生，不从PendingAction payload反推Model Projection；
- Tool Binding与Runtime Settlement fixture按公开exact request/ref复读，合法内容经公开Seal/Validate形成；
- 首次协调返回完整ToolResult、Inspection与Association闭包；
- 同一Request重放只复读既有结果，测试Tool执行计数保持一次；
- Binding current到期时在Tool执行前Fail Closed，执行计数为0；
- Association被替换为另一份内部合法、重新seal的内容时拒绝Application结果，Result Store保持空；
- 未调用Context Refresh、Harness Continuation或Turn推进。
