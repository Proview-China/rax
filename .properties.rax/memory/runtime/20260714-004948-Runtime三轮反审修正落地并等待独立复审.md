## 本次冻结的核心设计事实

1. V1使用`IdentityExecutionLease`保证一个`AgentIdentity`只有一个活跃执行Lineage；一个Lineage只绑定一个`ResolvedPlanDigest`；重建创建新`instance_id`并提升`instance_epoch`。
2. 实例状态由`LifecyclePhase`、`ExecutionCertainty`和`CleanupStatus`三维共同表达；不能把已发Stop、Harness自报或Sandbox释放写成已确认终止和完全清理。
3. `ExecutionFence`、`RevocationPolicy`、`ConflictEffectDomain`与`ReplacementPermit`共同约束失联旧实例和替换；离线Token不承诺即时撤销，具体撤销时延与时钟偏差必须显式配置和审核。
4. Harness按Conformance等级准入；不能把外部动作强制经过受治理效果边界的Harness不得宣称为受治理Agent。
5. 接纳拆分为Static Admission、Bounded Preflight和Activation；激活生成不可变`ActivationSnapshot`。组件绑定采用可恢复`BindingSaga`，补偿作为独立受权外部效果，不承诺跨系统原子回滚。
6. 统一Effect覆盖模型Context外发与费用、Sandbox、网络、Tool/MCP、Hosted Tool、Provider长期操作、Prompt Cache、Credential、Artifact/Memory提交和安全控制；使用`EffectIntent`、最终边界授权/Fence、`EffectReceipt`与`UnknownOutcome`，不承诺Exactly Once。
7. 预算被定义为`BudgetAuthorityPort`必要能力，不预设新增模块；Runtime不拥有价格语义。
8. 命令在单一线性化事实源排序，安全命令支配推进命令；分区时读取可降级，但推进、授权、续租和新Effect一律fail closed。
9. Evidence区分provenance、observation、attestation和authoritative fact；高风险Effect要求write-ahead权威Intent。Restricted Evidence与Public Projection分离，Harness自报不能成为Ready、Cleaned或Effect结算的唯一依据。
10. Checkpoint一致性、Cache安全分区、扩展供应链、正式Artifact/Memory提交、RemoteContinuation与Provider长期状态全部纳入恢复、Fence、清理和UnknownOutcome合同。
