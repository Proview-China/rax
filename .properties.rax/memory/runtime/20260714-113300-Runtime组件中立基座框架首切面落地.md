# Runtime组件中立基座框架首切面落地

## 事件

2026-07-14，用户明确授权按照已复审Runtime设计与V1 Plan开始写代码，同时要求Harness、Context/Cache、工具链、审批链及其他组件暂不实现，由后续用户逐项指导。Runtime在`ExecutionRuntime/runtime`建立独立Go module，现有`model-invoker`保持原状并通过未来版本化Port接入。

## 本次产物

1. `core`：Identity、Lineage、Instance、Lease、Run、Effect和Owner引用；三维状态与迁移；epoch/revision前置条件；TypedError；Fence、显式撤销策略、ReplacementPermit、统一Effect和受限恢复白名单；
2. `control`：Transport-neutral Command Envelope与安全命令支配；
3. `kernel`：线程安全的纯Runtime Instance Aggregate、revision线性化和迟到Observation分类；
4. `ports`：Harness、Model Invoker、Context/Cache、Tool/MCP、State、Governance、Sandbox、Budget与Evidence接入合同；
5. `tests`：非法三维状态、terminal不可复活、certainty收敛、旧epoch、Write-ahead Intent、离线撤销、恢复Effect、命令支配、并发revision和Port证据反例；
6. 中文实现README、模块说明和项目索引同步。

## 验证

- `go test ./...`：通过；
- `go test -race ./...`：通过；
- `go vet ./...`：通过。

## 保持边界

- 未修改`ExecutionRuntime/model-invoker`；
- 未实现Harness内部组成；
- 未实现Context/Cache、Tool/MCP、Review/Management/Organization、Memory/Knowledge/Asset或Sandbox内部能力；
- 未选择数据库、RPC、进程拓扑、生产Sandbox、真实账号、外部集成或SLA；
- 下一切面只推进Runtime自身的线性化事实Port、内存fake和无循环Admission/Activation。
