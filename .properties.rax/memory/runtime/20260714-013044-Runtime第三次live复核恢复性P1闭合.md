# Runtime第三次live复核恢复性P1闭合

## 事件

独立线程确认此前Fence所有者、Intent所有权、说明目录与两张图的修正方向正确，同时指出`unknown/lost/fenced`下恢复Effect允许范围和ExecutionCertainty收敛仍不可判定。本次继续只修Runtime设计、Plan、矩阵与项目状态，不进入实现。

## 形成的事实

1. 不确定态禁止一般业务Effect，但允许可判定的受限恢复白名单：Inspect、原Intent结算、Emergency Safety，以及具有新Intent/Fence/Authority/适用Budget/Write-ahead Evidence且不扩大权限的Cleanup/Release；
2. Compensation只有原效果已经由权威Receipt或Inspect确定并取得独立授权时允许；原效果仍Unknown时禁止猜测补偿；
3. `unknown/lost`只有在权威事实覆盖全部相关Effect、Lease与冲突域后才可原地收敛到`confirmed`；`fenced`旧Instance不能恢复运行；terminal状态不得复活Lifecycle；
4. 反例矩阵新增“Unknown原效果自动补偿”和“Cleanup因only inspect永久锁死”两个失败输入，Plan同步加入白名单和收敛验证任务；
5. 正式Plan门禁继续关闭，等待独立线程最终按文件复审。
