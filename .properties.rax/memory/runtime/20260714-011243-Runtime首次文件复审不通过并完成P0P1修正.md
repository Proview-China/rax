## P0修正

冻结唯一无循环对象建立协议：

```text
proposed Lineage
-> proposed Instance ID/epoch
-> Bounded Preflight（关联proposed Instance）
-> ActivationSnapshot（只绑定proposed身份与Sandbox需求）
-> reserved IdentityExecutionLease（排他但只允许激活/清理）
-> Budget Reservation（适用且存在有限cap时）
-> reserved_quarantined SandboxLease
-> 最终重验与ActivationCommit
-> active Identity/Lineage + provisioning Instance
-> active SandboxLease
```

每一步均定义proposed/reserved/active语义、允许权力和失败恢复；跨系统不承诺原子提交，未知Lease保留Quarantine、冲突域和残留证据。

## P1修正

1. Intent、Settlement、Review Verdict与Artifact/Memory Commit分别绑定唯一语义所有者；
2. 三维状态补齐完整合法谓词、迁移前置条件、触发、失败落点、迟到事件和重建规则；
3. 只有`leased_offline`且Token/Fence/Policy仍有效的已持久化Intent可在分区中继续，`online_strict`一律fail closed；
4. Checkpoint改用Effect Intent Accept、Dispatch、Settlement和RemoteContinuation水位，Action只保留为可选子维度；
5. Facade命令补齐Target ResourceRef、Identity/Lineage/Instance/Lease/Authority epoch、revision、幂等范围与稳定TypedError；
6. 统一Fence改用`effect_intent_id/revision`，ActionRef只作可选子类型引用；
7. 区分本地纯Cache查找、父模型Intent内Provider命中和独立远程Cache Effect，并要求命中返回前重鉴权；
8. Hard Budget只在存在可强制执行的有限最坏cap时成立，无界stream/batch必须拒绝或显式降级Soft；
9. 扩展合同改为实现中立的隔离属性与等价证明，不预选地址空间/进程拓扑；
10. Admission、对象、Effect/Evidence、Fence与Continuity五张图已按文字合同修正；Sandbox本地Release与远程Continuation结算明确拆分；
11. 反例矩阵补入Profile四例、事件重复/EventGap/Command逻辑提交，以及本轮所有新增失败例；三轮34条审计追溯同步更新；
