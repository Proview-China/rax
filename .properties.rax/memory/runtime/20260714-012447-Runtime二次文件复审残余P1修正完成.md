## 本次修正

1. Preflight成功或confirmed-not-applied不得留下Runtime清理义务；Probe结果未知或可能残留时，状态原子进入`stopping + unknown + pending/indeterminate`，不得停留preflighting或形成Snapshot；
2. ExecutionFence由每个部署绑定的唯一Fence/Authority epoch事实所有者拥有；Sandbox、网络、Invoker、Tool/MCP、Cache和Commit只是多个强制执行点；
3. Runtime只负责向绑定Intent Port提交、协调和关联EffectIntent，不自动拥有所有Intent语义；每个Effect仍解析到一个唯一Intent所有者；
4. 对象图改为AgentInstance分别连接Lifecycle、ExecutionCertainty和Cleanup三个正交维度，不再画成依赖链；
5. Continuity图把Review Accepted与Rejected拆成两条支路，只有Accepted可以进入Formal Commit；
6. Plan不再写死`module/runtime`，说明资产落点留待用户批准；
7. Snapshot只冻结Budget Policy/requested cap，真实Budget Reservation在reserved Identity Lease之后取得，避免新增预算循环；
8. 反例矩阵新增Preflight未知、Fence多执行点、Intent协调边界、三维正交、Rejected Commit和Plan目录等失败输入，三轮34条追溯同步更新。
