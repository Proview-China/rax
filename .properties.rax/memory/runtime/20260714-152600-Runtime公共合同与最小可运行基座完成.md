# Runtime公共合同与最小可运行基座完成

## 事件

2026-07-14，用户确认采用“先固定公共合同 → 做最小可运行骨架 → 组件逐个通过Port接入 → 每接一个跑完整闭环验收”的实施路径。本次完成Runtime组件中立基座；未接入任何真实Harness、Sandbox、Tool/MCP、Memory、Review或其他相邻组件实现。

## 本次产物

1. 固定Identity、Lineage、Instance、Run、Timeline、Checkpoint、Restore/Rewind、Effect、Fence和分维度终止合同；
2. 固定Profile/Assembly、Execution、Context、Tool/MCP、Memory/Knowledge/Asset、Organization/Review/Management、Sandbox、Budget、Evidence、Timeline和Checkpoint责任域Port；
3. `ComponentRegistry + ResolvedAgentPlan`验证组件版本、Digest、Capability TTL、依赖DAG、Required/Optional和Residual，依赖变成Residual后不会错误绑定上层组件；
4. Activation Journal逐步持久化`proposed → preflighting → preflight_passed → snapshot_frozen → identity_lease_reserved → budget_resolved → sandbox_reserved → committed → sandbox_active`；
5. `ActivationFactPort.CommitActivation`把ActivationAttempt、Instance的Sandbox绑定和Identity Lease active固定为一次逻辑提交；
6. Foundation Coordinator跑通Plan校验、Activation、独立Ready、单Run、consistent Checkpoint、以新Instance恢复、Stop、Fence、Release和分维度终止；
7. 两种Execution Conformance通过同一闭环；Ready证据冲突、Checkpoint partial、Allocate/Activate/Release回包丢失均fail closed；
8. `RecoverActivation`只从持久化Journal取得`PlanRecovery`的唯一安全动作；记录过Intent或结果未知时只Inspect，不重复派发。
9. Kernel增加显式`SupervisionPolicy`与纯函数决策：无默认生产SLA，运行中Policy Digest漂移被拒绝，Identity Lease续租优先于普通Inspect，连续失败有界退避后Quarantine，Authority撤销或Lease过期只会Fence并停止；10,000轮长期调度测试保持有界。

## 保持边界

- 目前所有相邻组件只有Port、fake或stub，没有提前实现领域算法；
- fake FactStore、Environment、Execution、Evidence和Checkpoint不是生产后端选择；
- 没有选择数据库、RPC、进程拓扑、真实Sandbox、远程API、SDK或SLA；
- Checkpoint Restore/Rewind只创建新Instance ID与更高epoch，不复活旧实例，也不承诺回滚外部世界；
- 下一步从单个组件开始：先确认其Port/Descriptor/Capability与Effect边界，再重复完整闭环验收。
