# Runtime P0.6公共治理入口与Application边界冻结

## 事件

2026-07-15，Runtime在P0.5 Run Settlement基础上完成P0.6所需公共治理入口、只读恢复面与Application import边界。Runtime范围普通测试、shuffle Race与Vet由协作线程独立全量验证通过；本事件不等同于Application/Harness最终组合闭环或6+1解锁。

## 已冻结公共入口

- `OperationEffectAdmissionPortV3`：受绑定Intent Owner的proposed→accepted准入和accepted事实只读恢复；
- `OperationGovernancePortV3`：完整Permit+Fence授权Envelope的Issue、Begin、unknown与Inspect；
- `ExecutionDelegationGovernancePortV2`：Host relay→Data Provider的declare、prepared commit与Inspect；
- `OperationObservationGovernancePortV3`：精确prepared attempt、source sequence与Evidence Record绑定的Observation摄取；
- `OperationSettlementGovernancePortV3`：Settlement Owner提交封闭Disposition、Evidence与Schema+Digest领域结果；
- `RunLifecycleGovernancePortV3`与`RunStartGovernancePortV3`：pending Run创建、governed start确认、停止、Settlement、CompleteRun与终态Termination恢复；
- `RunClaimIngestGovernancePortV3`：certified Run历史认证preflight后，精确摄取/Inspect Claim Evidence Ledger Record与create-once Association；Claim不选择Outcome；
- `RunSettlementPlanAdmissionPortV3`：聚合current BindingSet全member Fact/grant、宿主baseline、唯一certifier与每个组件显式声明，产生create-once Plan Certification；
- `ApplicationCommandFactPortV2`：Command、Desired State与Outbox的唯一Application公共事实面。

## Owner与恢复边界

- Application只允许导入`runtime/core`和`runtime/ports`；`control/kernel/foundation/fakes`均为Runtime内部实现，`control` Command别名仅供legacy/restricted迁移；
- Host只relay，Data Provider在实际触达前独立复读Permit、Enforcement、Delegation及全部current治理事实；
- Provider Prepare/Execute回包丢失只Inspect预分配attempt，Inspect不得再次Execute；
- Observation不能直接成为Settlement、Harness terminal或Runtime Outcome；Settlement精确引用Observation或单层unknown Inspect provenance；
- Run先持久pending，governed start独立证明后才写running/StartedAt；terminal后的cleanup unknown只Inspect，Termination Progress可持续交付，Report不早产。
- Run start confirmation与pending→running由同一Run Fact Owner原子提交；公共Confirm/Inspect返回包含Operation digest、完整attempt/settlement refs、running revision与StartedAt的不可变Envelope，不能以裸running或相同StartedAt冒充另一attempt。

## 保留限制

- Application Step Journal、Command Outbox消费与Harness governed bridge仍由独立任务完成最终组合接线；
- 当前仍不可宣布6+1并行解锁；必须等待Runtime/Application/Harness组合普通、Race、Vet与故障门禁完成；
- 未选择生产数据库、RPC、Scheduler、签名、进程拓扑、跨Owner事务模型或SLA；
- fake与Conformance只证明合同、CAS和恢复语义，不声明production durability或availability。
- `CreatePendingRunV3`已强制Run+Plan+Certification Association同Owner create-once；首次写入要求current认证，丢回包与重启通过immutable历史Certification恢复。组件只能声明，不能certify/create；可信Assembler资格不会下放给普通Application业务代码。
- Binding Plan身份已改为完整Requirement canonical派生：Requirements/Capabilities集合规范化，任一Kind、Artifact、Contract、Capability或required语义替换都会使旧PlanDigest失效；Adapter依赖扫描通过`CheckAdapterRuntimeImportsV2`执行，只允许`runtime/core`与`runtime/ports`且不授任何生产资格。
- Runtime公共入口在触达Fact/Evidence/backend前先校验Envelope、scope/ID/revision与依赖；Begin、unknown、Observation、Settlement、Plan Admission、Run Start/Lifecycle/Claim的无效输入均有spy零读取反例。Provider的64并发逻辑幂等仅存在于isolated fixture profile，分别报告method entries、distinct observations与logical effects；它不宣称物理exactly-once，ProductionClaimEligible始终为false。
