# Runtime持久化Run事实与崩溃恢复落地

- 时间：2026-07-14 16:06（Asia/Shanghai）
- 模块：Runtime
- 事件：在首个Harness通过Foundation闭环后，补齐24×7运行所需的Runtime Run事实持久化切面。

## 已完成

1. 新增`control.RunFactPort`、CAS请求与持久化迁移验证，Run Record继续由Runtime唯一拥有；
2. 内存FactStore实现单Instance单活跃Run的原子门禁、Run revision CAS和活跃索引；
3. 新增`kernel.RunJournal`，提供Start、BeginStop、Finish、Inspect和RecoverActive；
4. 启动或终态写入已提交但回包丢失时，新协调器只凭持久事实Inspect恢复，不重复创建或覆盖；
5. 相同终态结算幂等，不同终态并发Claim只允许一个结果线性化，迟到冲突不能改写首个Runtime事实；
6. 未修改`ExecutionPort`，未解释Harness Opaque Payload的领域语义，也未把Harness completion自动提升为Runtime权威终态。

随后在相同边界下补齐Claim摄取：

7. 新增通用Opaque Payload内容摘要验证和`kernel.RunClaimGateway`；
8. 从Harness terminal Event取得source sequence后，Runtime先验证Instance epoch与Payload Digest，再幂等写入Evidence；
9. 同一source identity/epoch/sequence重放返回原Evidence引用，换内容则产生`evidence_conflict`；
10. 合法Observation只关联不可变`RunCompletionClaim`，Run仍保持原状态且`ExecutionOutcome`为空。
