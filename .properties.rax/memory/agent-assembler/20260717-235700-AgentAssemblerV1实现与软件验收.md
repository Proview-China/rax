# Agent Assembler V1 实现与软件验收

## 事件

2026-07-17，Agent Assembler V1 在已确认设计和 Plan 边界内完成 owner-local 实现与定向软件验收。

## 落地产物

- `contract`：Component Release/Catalog、Resolution Facts、Resolved Agent Plan、current projection 合同和 canonical seal；
- `ports`：Assembler、exact readers、plan repository、Owner release publisher/reader；
- `resolver`：Definition + frozen Facts/Catalog 的确定性 production 解析；
- `mapper`：Runtime `BindingPlanV2` 与 Harness `AssemblyInputV1`；
- `repository`：revision-aware exact snapshots、create-once plan/release、exact current CAS 和历史 Plan ABA 防护；
- `conformance` 与测试：七核心正向、自定义组件、optional prune、S1/S2 drift、TTL、lost reply、并发、import boundary、Harness 实编译。

## 关键裁决

- Catalog release identity 使用结构化 `(release_id, revision)`，不做字符串拼接；
- Catalog checked 时刻之前必须已创建 release，且 release 在 checked 时刻仍有效；
- current pointer 本身有独立 revision/digest，CAS 比较 exact current projection，而不是只比较 PlanRef；
- current revision 单调递增，并拒绝退回任一历史 PlanRef；
- Resolve 的 CAS 回包丢失只接受 exact intended current projection 的 Inspect 结果；
- Assembler 不授予 activation、authority、dispatch，不触碰 Provider/secret/Sandbox/Runtime execution。
