# Agent Assembler 设计骨架

## 设计状态

- 当前阶段：设计草案。
- 实现授权：无；本文件只固定讨论边界。
- 当前共识：装配完成后产出不可变 `ResolvedAgentPlan`。
- 模块原则：装配器独立于 Runtime，也独立于各能力引擎。

## 作用

Agent Assembler 把已批准的 `AgentDefinition` 解析为可执行前检查的装配计划。
它负责证明所引用组件能够形成一个一致组合，而不是启动 Agent。
它类似镜像构建清单解析器，但不会把外部引擎复制进自身。

## 输入

- 一个确定版本的 `AgentDefinition`。
- Harness、Profile、上下文、工具、MCP、记忆、知识和策略的能力描述。
- 组件注册表、版本与兼容性约束、目标 Runtime 能力、沙箱需求和组织权限引用。

## 输出

- 版本完全固定、内容可摘要的 `ResolvedAgentPlan`。
- 组件绑定表、启动前置条件和所需 Sandbox 能力集合。
- 解析证据、兼容性报告、风险与拒绝原因。
- 无法满足约束时的确定性失败，不执行隐式降级。

## 明确拥有

- 依赖解析、版本固定和组件兼容性检查。
- Definition 到 Resolved Plan 的确定性转换规则。
- 装配计划摘要、来源链和可复现性证据。
- 对缺失、不兼容、未获授权组件的装配拒绝。

## 明确不拥有

- 不拥有任何组件内部语义或实现生命周期。
- 不启动 Harness，不申请沙箱，不执行 Agent Run。
- 不调度节点，不管理实例健康，不进行失败重建。
- 不替代审核引擎作业务判断，也不扩大 Agent 权限。
- 不为通过装配而改写提示词、模型参数或工具意图。

## 与 Runtime 的关系

Assembler 是 Runtime 的上游生产者，Runtime 是 Resolved Plan 的消费者。
Runtime 只能验证并绑定计划，不得在启动时重新解释 Definition。
计划摘要应成为实例、Run、事件和审计记录的共同关联键。
身份保持持久；同一计划可产生可替换实例，每个实例可承载多个短暂 Run。

## 依赖

- Agent Definition 的稳定 Schema 和版本规则。
- 各模块发布的能力清单、兼容矩阵与完整性证据。
- 策略、权限、资产、沙箱能力注册表，以及 Runtime 提供的目标能力声明。

## 待共同决定

- Resolved Plan 的字段、签名、摘要和有效期模型。
- 解析是完全离线、受控联网还是两阶段完成。
- 可选能力缺失时是拒绝、显式降级还是产生条件计划。
- 多平台 Sandbox 后端的能力匹配表达方式。

## 进入计划阶段的门槛

- [ ] Definition 到 Plan 的解析流程和失败路径获得确认。
- [ ] 组件能力描述协议与兼容判定规则明确。
- [ ] 不可变性、可复现性和供应链验证标准明确。
- [ ] Runtime 交接合同及其黑盒验收样例完成。
- [ ] 用户确认装配产物、技术职责、测试矩阵和实现范围。
