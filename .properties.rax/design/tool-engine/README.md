# 工具引擎设计骨架

## 设计状态

- 模块名称：`tool-engine`
- 当前阶段：设计草案，仅用于共同讨论模块边界。
- 实现授权：无；不得据此创建代码、服务、配置或计划。
- 独立性：本模块独立设计，通过显式契约与 Runtime 和其他引擎协作。

## 作用

工具引擎负责把异构工具整理为可发现、可调用、可取消、可审计的受控能力面。
它保护的是工具执行边界，不改变模型推理，也不替模型选择工具。
任何可能写文件、访问网络、修改外部系统或产生费用的调用，都属于外部效果。
外部效果默认不得执行，只有携带有效授权并通过运行时约束后才能发生。

## 输入与输出

### 输入

- 工具定义、版本、能力声明、参数 Schema 和作用域标识；
- Agent、Instance、Run、调用者标识和授权凭证；
- 工具调用请求、超时、取消和幂等信息；
- Runtime 提供的沙箱、网络、资源约束和授权凭证。

### 输出

- 可供上层发现的工具目录、能力描述和标准化调用结果；
- 错误、流式、取消、重试和幂等状态；
- 外部效果回执、资源使用记录、结果可信状态和审计事件。

## 明确拥有

- 工具描述、注册、版本、调用协议及结果封装的统一边界；
- 外部效果授权凭证的强制校验与效果回执；
- 工具级超时、取消、并发限制和错误分类。

## 明确不拥有

- 模型推理、工具选择和提示词工程；
- 职权分配、审核结论或授权策略本身；
- Agent 生命周期、集群调度、沙箱实现、MCP、记忆、知识和资产语义。

## 与 Runtime 的关系

Runtime 在实例装配时绑定工具能力，在 Run 内提交调用并传播取消信号。
工具引擎验证调用契约和授权，执行或转交工具，再返回标准事件与效果回执。
Runtime 拥有实例状态和资源租约；工具引擎不反向控制 Runtime 生命周期。
两者只通过版本化接口协作，不要求与 Runtime 部署在同一进程或同一沙箱。

## 依赖

- Agent 身份、职责、职权和可验证授权凭证；
- 沙箱和受控网络出口提供的执行边界；
- 审计事件、密钥引用和资产引用的公共契约。

## 待共同决定

- 工具能力与外部效果的分类体系；
- 授权凭证的签发者、粒度、时效和撤销方式；
- 幂等、重试、补偿、超时和流式返回语义；
- 本地工具、远程工具、Sidecar 的统一程度及结果可信度字段。

## 进入计划阶段的门槛

- 作用、拥有边界和不拥有边界获得共同确认；
- 工具调用时序图、核心 Schema、错误模型和兼容规则完成；
- 无授权、有授权、授权撤销和部分失败路径可验证；
- 单元、白盒、黑盒和 Runtime 集成验收矩阵完成；
- 用户明确批准从设计进入计划阶段。
