# 知识引擎设计骨架

## 设计状态

- 模块名称：`knowledge-engine`
- 当前阶段：设计草案，只建立知识能力的独立边界。
- 实现授权：无；不得据此创建采集管线、索引、服务或计划。
- 独立性：本模块独立于记忆引擎、上下文引擎、资产管理器和 Runtime。

## 作用

知识引擎负责管理经过来源标识、版本治理和可信度描述的可检索知识。
知识是可复用、可验证的前置认知材料，不等同于某次 Run 的经验记忆。
正式知识快照、来源和索引必须位于可销毁沙箱之外。
沙箱内只能挂载获准快照或缓存，不能静默改写正式知识。

## 输入与输出

### 输入

- 知识包、来源、版本及解析、索引、更新和撤回请求；
- 查询、范围、时效、可信等级和访问授权约束；
- 来源变更、冲突、失效或人工审核结果；
- Runtime 绑定的 KnowledgeSnapshotRef 和访问授权。

### 输出

- 带来源、版本、可信等级及差异状态的知识候选；
- KnowledgeRef、快照、处理结果、引用证据和审计事件。

## 明确拥有

- 知识包、来源、版本、快照和可信状态模型；
- 采集、规范化、索引、更新、撤回、来源追溯和查询契约；
- 正式知识持久化与沙箱外隔离要求。

## 明确不拥有

- 某次 Run 的经验记忆和用户会话状态；
- 原始正式资产的字节存储、所有权与版本治理；
- 提示词、模型判断、Runtime 生命周期、沙箱、调度或授权策略。

## 与 Runtime 的关系

Runtime 在实例装配时绑定明确版本的 KnowledgeSnapshotRef 和访问范围。
知识引擎按查询契约返回候选与证据，不接管 Run 状态或模型上下文。
Runtime 可传播快照失效和取消信号，但不能直接改写正式知识。
实例销毁和迁移只影响挂载，不影响沙箱外的正式知识及其来源链。

## 依赖

- 资产管理器提供稳定来源引用和内容读取能力；
- 身份、授权、审核、审计、持久存储和检索基础设施；
- 上下文引擎的证据消费和引用格式。

## 待共同决定

- 知识包、快照、来源和可信等级的最小 Schema；
- 采集方式、更新频率、失效检测和撤回机制；
- 多来源冲突、版本分叉和权威来源优先级；
- 多层可见性、继承、检索排序、引用粒度和时效指标。

## 进入计划阶段的门槛

- 知识、记忆、资产和上下文边界获得共同确认；
- 关键时序图与 KnowledgeRef、来源、版本、可信状态 Schema 完成；
- 沙箱销毁、来源失效、冲突和越权查询路径可验证；
- 单元、白盒、黑盒、来源追溯和隔离测试矩阵完成；
- 用户明确批准从设计进入计划阶段。
