# MCP 网关设计骨架

## 设计状态

- 模块名称：`mcp-gateway`
- 当前阶段：设计草案，仅固定首轮讨论对象。
- 实现授权：无；不得据此创建实现、连接真实服务或进入计划。
- 独立性：本模块是独立协议边界，不并入工具引擎或 Runtime 内核。

## 作用

MCP 网关负责连接、发现和管理符合 MCP 契约的外部能力提供方。
它把不同传输、会话和服务端生命周期转换为稳定的内部访问面。
网关必须保留标准 MCP 兼容性；扩展能力只能作为可选层另行确认。
MCP 工具调用可能产生网络、文件、费用或第三方状态变化，均视为外部效果。
任何外部效果都必须获得明确授权；连接成功或能力可见不等于允许执行。

## 输入与输出

### 输入

- MCP Server 引用、启动/连接描述和可信来源信息；
- 传输、会话、认证、超时、资源限制与能力请求；
- Agent、Instance、Run 上下文及有效授权凭证；
- Runtime 发出的绑定、取消、释放、健康检查信号和授权凭证。

### 输出

- 标准化能力目录、会话状态、健康状态和版本信息；
- 调用结果、协议错误、取消、效果回执和审计事件。

## 明确拥有

- MCP 连接、传输、会话、生命周期适配和能力快照；
- MCP 请求与内部受控调用契约之间的转换；
- 调用前授权门禁及协议级审计证据。

## 明确不拥有

- MCP Server 提供能力的业务语义和正确性；
- 工具权限、组织职权或审核结论的制定；
- Agent 推理、提示词、记忆、知识、资产、Runtime 状态或沙箱调度。

## 与 Runtime 的关系

Runtime 为 AgentInstance 绑定经批准的 MCP Server 引用和能力快照。
网关维护对应连接与会话，并把健康、变更、错误和调用事件回传 Runtime。
Runtime 负责 Run 生命周期和取消传播；网关负责协议范围内的安全收敛。
网关可以独立部署，其故障不得破坏 Runtime 对其他能力模块的控制。

## 依赖

- MCP 标准协议与版本兼容基线；
- 身份、职权、授权、审核凭证、密钥引用和网络出口；
- 工具引擎的通用效果模型和统一审计事件契约。

## 待共同决定

- 首批支持的传输、认证和 Server 托管模式；
- Server 信任分级、能力缓存和动态刷新语义；
- 标准 MCP 与可选扩展层的严格分界；
- 会话复用、隔离、恢复、背压及各类能力的授权粒度。

## 进入计划阶段的门槛

- 协议边界、工具引擎边界和 Runtime 边界共同确认；
- 关键时序图、Server 信任模型、快照和门禁 Schema 完成；
- 恶意 Server、能力漂移、断连和取消场景具备验收设计；
- 标准兼容、黑白盒和 Runtime 集成测试矩阵完成；
- 用户明确批准从设计进入计划阶段。
