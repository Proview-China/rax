# 记忆引擎设计骨架

## 设计状态

- 模块名称：`memory-engine`
- 当前阶段：设计草案，只建立概念和责任边界。
- 实现授权：无；不得据此创建存储、索引、服务或计划。
- 独立性：本模块独立于上下文引擎、知识引擎、资产管理器和 Runtime。

## 作用

记忆引擎负责保存 Agent 经历形成的、可追溯且可治理的长期状态。
记忆属于持久身份或作用域，不属于某个可销毁的 AgentInstance 进程。
正式记忆及其索引必须位于可销毁沙箱之外；沙箱内只能存在受控缓存或副本。
它向上下文引擎提供候选记忆，但不决定最终放入模型上下文的内容。

## 输入与输出

### 输入

- 稳定身份引用及事件、观察、结果和待写入记忆候选；
- 来源、时间、置信度、敏感级别和保留策略；
- 检索请求、查询约束、作用域和访问授权；
- 更正、遗忘、删除、归档、合并请求和访问授权。

### 输出

- 带来源、版本、置信度、作用域及写入状态的候选记忆；
- MemoryRef、变更、审计及检查点恢复所需的状态引用。

## 明确拥有

- 记忆记录、作用域、来源、版本和生命周期模型；
- 写入门禁、去重、合并、更正、遗忘、保留与检索索引；
- 持久记忆的隔离、完整性和审计证据。

## 明确不拥有

- 模型推理、提示词组装和上下文预算分配；
- 经正式治理的知识事实或正式资产本体；
- Agent 治理、Runtime 生命周期、调度、沙箱和检查点总编排。

## 与 Runtime 的关系

Runtime 在装配 AgentInstance 时绑定可访问的 MemoryScope 与版本约束。
Run 过程中产生的记忆先作为候选提交，不能因模型输出而直接成为永久事实。
Runtime 只传递身份、事件和生命周期信号，不拥有记忆内容及持久存储。
沙箱销毁、实例重建或迁移不得导致已确认记忆丢失或越权泄漏。

## 依赖

- 稳定身份、访问授权、持久存储、索引、加密和审计；
- 上下文引擎的检索请求与候选消费契约；
- Runtime 的 Run、事件、检查点和恢复引用。

## 待共同决定

- 工作记忆、情景记忆、关系记忆等分类是否进入核心；
- 候选记忆晋升、人工确认和自动衰减规则；
- 作用域分享、组织隔离及删除、冲突、过期和隐私语义；
- 检索排序、快照一致性和容量配额边界。

## 进入计划阶段的门槛

- 记忆与上下文、知识、资产的边界获得共同确认；
- 关键时序图与 MemoryRef、作用域、来源和变更 Schema 完成；
- 沙箱销毁、实例迁移、越权读取和删除路径可验证；
- 单元、白盒、黑盒、恢复和隔离测试矩阵完成；
- 用户明确批准从设计进入计划阶段。
