# 资产管理器设计骨架

## 设计状态

- 模块名称：`asset-manager`
- 当前阶段：设计草案，只建立正式资产的治理边界。
- 实现授权：无；不得据此创建存储服务、迁移数据或进入计划。
- 独立性：本模块独立于 Runtime、沙箱、记忆引擎和知识引擎。

## 作用

资产管理器负责保存可寻址、可校验、可版本化和可追溯的正式资产。
正式资产包括经确认的输入材料、配置资产、成果和其他受治理对象。
正式资产及其元数据必须位于可销毁沙箱之外，不能以沙箱文件系统为事实源。
沙箱内文件默认只是临时工作副本，只有经过显式晋升流程才能成为正式资产。

## 输入与输出

### 输入

- 待登记内容、引用、元数据、来源及全生命周期请求；
- Agent、Instance、Run、租户和组织作用域；
- 访问授权、完整性要求、保留策略和敏感等级；
- 沙箱产物、效果回执、校验摘要、验收状态和访问授权。

### 输出

- 稳定 AssetRef、版本、摘要、来源链及受控读取结果；
- 晋升、派生、拒绝、归档、删除、完整性和审计结果。

## 明确拥有

- 正式资产命名、寻址、版本、摘要和来源链模型；
- 资产登记、读取、派生、晋升、归档、保留和完整性边界；
- 临时产物成为正式资产时的显式状态转换。

## 明确不拥有

- 沙箱内部临时文件系统和进程生命周期；
- 记忆内容、知识语义、提示词或模型推理；
- 成果审核、授权决策、组织职权、Runtime 状态和存储实现。

## 与 Runtime 的关系

Runtime 使用 AssetRef 装配输入，并把受控副本或只读挂载交给沙箱。
Run 产物先保持临时状态；Runtime 只能提交晋升请求，不能静默写入正式资产。
资产管理器返回稳定引用与审计结果，不接管 AgentInstance 生命周期。
沙箱销毁、实例重建和节点迁移不得影响正式资产的完整性与可追溯性。

## 依赖

- 身份、职权、授权、持久存储、摘要和加密能力；
- 沙箱文件交换、挂载和结果收集契约；
- 记忆与知识引擎对 AssetRef 的稳定引用规则。

## 待共同决定

- 哪些对象属于正式资产以及 AssetRef 的最小 Schema；
- 不可变版本、可变别名、派生关系和并发写入语义；
- 沙箱产物的晋升、验收、拒绝和回滚流程；
- 删除、归档、配额、隔离及大资产传输和校验指标。

## 进入计划阶段的门槛

- 正式资产、临时产物、记忆和知识边界获得共同确认；
- 关键时序图与 AssetRef、版本、来源链、状态和授权 Schema 完成；
- 沙箱销毁、晋升失败、并发冲突和越权访问路径可验证；
- 单元、白盒、黑盒、完整性和 Runtime 集成测试矩阵完成；
- 用户明确批准从设计进入计划阶段。
